Datos compuestos

Hasta el momento hemos utilizado datos primitivos: number, string, boolean, undefined, null.
Ahora vamos a trabajar los datos compuestos: object={}, array=[], funcion(), class={}, etc.

primero empezemos con las funciones, a las funciones se les puede pasar parametros , como en el siguiente ejemplo: 
            
                //definir funcion
                function saludar(nombre, edad){
                    console.log(`mi nombre es ${nombre} y tengo ${edad} años `)
                }

                //invocar a la funcion
                saludar("Rodney Zapata",47);
En el case que se invocar la funcion solamente sin pasarles parametros, entonces Javasript colocara los valores como indefinided, Tambien se puede colocar valores por defectos a partir del 2015 y es de la siguiente forma: 
            
                
                    //definir funcion con valores de parametros por defecto
                    function saludar(nombre="", edad=0){
                        console.log(`mi nombre es ${nombre} y tengo ${edad} años `)
                    }
existe otro tipo de funcion llamada anonima, donde es una funcion dinamica donde no se conoce el nombre de la funcion ni el valor de la variable.
Tambien primero hay que definirla para despues ser invocadas en caso contrario genera error, en cambio las funciones anterior declarada pueden ser invocadas en cualquier parte del codigo inclusive antes de ser definida.

            
                //funcion expresada o anonima
                const funcionExpresion=function(){
                    console.log("funcion anonima, no se conoce el nombre de la funcion ni el valor de las variables");
                }
        
                //invocamos la funcion
                funcionExpresion();

    promesas

    Las promesas se usan en llamadas asincronicas que por lo general demoran y nos pueden arrojar un resultaco exitoso o fracaso.
    Esencialmente, una promesa es un objeto devuelto al cual se adjuntan funciones callback, en lugar de pasar callbacks a una función.
    Una promesa tiene 3 estados:

    Pending (pendiente) → cuando la operación aún no terminó.
    resolved (resuelta) → cuando la operación terminó con éxito y devuelve un valor.
    Rejected (rechazada) → cuando la operación falló y devuelve un error.

    📌 Métodos útiles

    then() → se ejecuta si la promesa se resuelve bien.
    catch() → se ejecuta si la promesa falla.
    finally() → se ejecuta siempre (éxito o error).

    ✅ Conclusión:

    Si usas .then().catch() → no necesitas try...catch.
    Si usas async/await → debes usar try...catch. 
                

                //Codigo en Javascript
                async function getUser() {
                    try {
                        const promesa = fetch("https://jsonplaceholder.typicode.com/users")
                        const data = (await promesa).json;
                        console.log("usuarios", data)
                
                    } catch (error) {
                        console.log("error : ", error);
                    }
                
                }
                
                //desde React codigo clasico
                useEffect(() => {
                  fetch("https://jsonplaceholder.typicode.com/users")
                    .then(response => response.json())
                    .then(data => setUsuarios(data))
                    .catch(err => setError(err.message));
                }, []);
                
                //desde React nueva forma usando TryCatch
                useEffect(() => {
                  const cargarUsuarios = async () => {
                    try {
                      const response = await fetch("https://jsonplaceholder.typicode.com/users");
                      const data = await response.json();
                      setUsuarios(data);
                    } catch (error) {
                      setError(error.message);
                    }
                  };

    manejo de errores

    El manejo de excepciones en JavaScript sirve para controlar errores que pueden ocurrir en tiempo de ejecución, evitando que el programa se detenga abruptamente.
    Se hace principalmente con try...catch...finally, pero también existen patrones modernos para manejar errores en código asíncrono con Promises y async/await. 

                
                
                function dividir(a, b) {
                    if (b === 0) {
                        throw new Error("Error al dividir entre cero. ");
                    }
                    return a / b;
                }

                // para probarlos usamos un try/casth
                
                try {
                    console.log(dividir(8, 0));
                } catch (error) {
                    console.log(error.message)
                }

    modulos

    En la forma antigua se debia export los modulos de la siguiente forma usando module.export{suma, potencia }
    y en la importacion usamos const moduloMatematica = require("./matematicas.js") 
                
                //archivo matematicas.js
                function suma(a, b) {
                    return a + b;
                }
                
                function potencia(a, b) {
                    return a ** b;
                }
                
                
                module.exports = {
                    suma,
                    potencia
                }
    y lo importamos en calcular.js 
                
                //archivo calcular.js
                const moduloMatematica= require("./matematicas.js")
                
                console.log(moduloMatematica.suma(2, 3));
                console.log(moduloMatematica.potencia(5, 2));
    Ahora la forma moderna para exporta e importar en la forma moderna a partir del 2016 ES6 es la siguiente:
    le antemponemos antes del nombre de la funcion la palabra export. 
            
            
            export function suma(a, b) {
                return a + b;
            }
            
            export function potencia(a, b) {
                return a ** b;
            }
            
            export const nombre = "matematicas avanzada";
    Esta funciones tambien lo podemos escribir en forma de funcion flecha de la siguiente forma: 
            
            
            export const getnombre = () => {
                return "Miguel de cervantes"
            }
            
            export const suma2 = (a, b) => {
                return a + b
            }
    y para importarlo lo usamos con las siguientes forma:
    import { suma, potencia, nombre } from "./matematicas.js"; desectructuramos los modulos que necesitamos. 
            

            import { suma, potencia, nombre } from "./matematicas.js";
            
            console.log(suma(2, 3));
            console.log(potencia(5, 2));
